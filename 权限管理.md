# 通用权限管理服务

## 功能

- 集中管理多个环境的权限；
- 支持多个对象对应多个角色；
- 支持多个角色对应对多个策略；
- 通过策略的属性来精确控制权限；
- 为了简单，不支持角色树，权限继承可通过策略实现；

## 基本概念

### 1. 环境

整个权限服务针对的某个特定的场景，可以是一个产品（应用权限管理）、一个数据库（数据库权限管理）或一个业务系统（系统权限管理）。

### 2. 对象

代表了环境里的一个用户，相同环境里的用户名唯一。

### 3. 角色

抽象的概念，往往会将多个角色指定给一个用户。

### 4. 策略

权限管理的核心，一个策略是一个申明，定义了以下：

- 资源：环境里需要被权限管理的物体；
- 属性：资源的某个子集；
- 操作：对于资源的某种动作，比如读、写；
- 效果：允许或拒绝

### 5. 数据

查询权限时的某种未知属性

## 权限模型

在某个环境里，存在多个对象，每个对象被指定了多个角色，每个角色赋予了多个策略；

每个策略定义了权限的具体内容，即允许或拒绝对某个资源的某个属性进行某种操作；

因此每个对象都最终拥有多个策略，这些策略的合集定义了该对象的权限；

## 命名规则

### 环境名，对象名，角色名，资源名，操作名

字符串。只包括`a-z`、`A-Z`、`0-9`、下划线`_`和点 `.`。

### 属性名

字符串数组。每个字符串需满足以下任一：

- 只包括`a-z`、`A-Z`、`0-9`、下划线`_`和点 `.`；
- 以感叹号`!`开始，后面的字符串只包括`a-z`、`A-Z`、`0-9`、下划线`_`和点 `.`；
- 星号`*`；

### 效果名

字符串枚举值，允许"ALLOW"或拒绝"DENY"。

### 数据名

字符串数组，每项需满足以下任一：

- 只包括`a-z`、`A-Z`、`0-9`、下划线`_`和点 `.`；
- 星号`*`；

## 对象-角色-策略的关系

对象与角色是多对多的关系。一个对象可以指定多个角色。多个对象可以被指定有同一个角色。

角色与策略是多对多的关系。一个角色可以赋予多个策略。多个角色可以被赋予同一个策略。

## 匹配规则

策略里的属性会在权限查询时与数据进行匹配。

### 规则一 单个数据与单个属性的匹配

- 属性为星号、\*时，匹配任何数据；
- 属性为！a 时，匹配任何非 a 的数据；
- 属性为 a 时，匹配 a 数据；

### 规则二 单个数据与策略里多个属性的匹配

单个数据只要与策略里的某个属性，按照规则 1 匹配了，该数据即与该策略匹配

注意

- 如果该策略的属性数组包含星号`*`，则整个属性等同于星号`*`；
- 如果数据数组为空，则数据数组等同于星号`*`；

### 规则三 单个数据与多个策略的匹配

如果对象的角色有多个策略，则单个数据会与每个策略进行匹配

- 首先拒绝策略会匹配，如果匹配成功则结果为拒绝；
- 然后允许策略会匹配，如果匹配成功则结果为允许；
- 如果以上都不成功，则说明权限为定义，返回 null；

### 案例

假设资源是"my_database"，操作是"read"，查询权限的数据是"a"，

```text
策略：
      资源 = "another_database"
      属性 = ["*"]
      操作 = "write"
      效果 = "ALLOW"
返回：
      null
理由：
      资源与操作完全不一样，该策略完全无关
```

```text
策略：
      资源 = "my_database"
      属性 = ["*"]
      操作 = "read"
      效果 = "ALLOW"
返回：
      允许
理由：
      匹配
```

```text
策略：
      资源 = "my_database"
      属性 = ["*", "b", "!c"]
      操作 = "read"
      效果 = "ALLOW"
返回：
      允许
理由：
      属性含星号，匹配 a
```

```text
策略：
      资源 = "my_database"
      属性 = ["!a", "b", "c"]
      操作 = "read"
      效果 = "ALLOW"
返回：
      null
理由：
      该策略未匹配 a，也没有拒绝策略，规则三最后一个
```

```text
策略：
      资源 = "my_database"
      属性 = ["b","d"]
      操作 = "read"
      效果 = "ALLOW"
策略：
      资源 = "my_database"
      属性 = ["a"]
      操作 = "read"
      效果 = "DENY"
返回：
      拒绝
理由：
      拒绝策略匹配
```

```text
策略：
      资源 = "my_database"
      属性 = ["*"]
      操作 = "read"
      效果 = "ALLOW"
策略：
      资源 = "my_database"
      属性 = ["!a"]
      操作 = "read"
      效果 = "DENY"
策略：
      资源 = "my_database"
      属性 = ["b","c"]
      操作 = "read"
      效果 = "DENY"
返回：
      ALLOW
理由：
      两个拒绝策略都未匹配，允许策略匹配成功。在实际应用应避免这种有歧义的定义
```
